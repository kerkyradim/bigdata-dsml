| Ενότητα εργασίας | Εργαλείο | Έκταση χρήσης | Τι έγινε χωρίς LLM |
|---|---|---|---|
| **Query 1 — κώδικας** (`Q1_df.py`, `Q1_df_udf.py`, `Q1_rdd.py`) | Βοηθητικό εργαλείο τεχνητής νοημοσύνης (chat-based coding assistant) | Βοήθησε στη δημιουργία του **βασικού σκελετού** των scripts (δόμηση αρχείου, `argparse`, `perf_counter`, paths HDFS, βασική ροή read → filter → aggregate → write) | Η **λογική του Query 1** (στήλες `Premis Desc` / `TIME OCC`, κανόνες τμημάτων ημέρας, υπολογισμός ποσοστών), οι **διορθώσεις**, οι **εκτελέσεις στο cluster**, η **επαλήθευση αποτελεσμάτων** και η **ανάλυση χρόνων** |
| **Query 2 — κώδικας** (`Q2_df.py`, `Q2_sql.py`) | Ίδιο εργαλείο | Βοήθησε στον **αρχικό σκελετό** (SparkSession, ανάγνωση CSV, temp views, timing, αποθήκευση αποτελεσμάτων) | Ο **σχεδιασμός του Query 2** (parsing `DATE OCC`, `row_number` ανά έτος, απλοποίηση SQL σε ξεχωριστά βήματα), οι **δοκιμές**, τα **αποτελέσματα** και η **σύγκριση επιδόσεων** |
| **Αναφορά — Query 1 & Query 2** (`report_queries_1_2.md`) | Ίδιο εργαλείο | Βοήθησε στη **διαμόρφωση κειμένου** και στη δομή ενοτήτων (πίνακες, συμπεράσματα) με βάση τα **δικά μου μετρημένα δεδομένα** | Η **τελική επεξεργασία**, η **επιβεβαίωση αριθμών** από τα log files, και η **προσωπική ερμηνεία** των αποτελεσμάτων |
| **Setup περιβάλλοντος (WSL, Spark, VPN, kubectl)** | Ίδιο εργαλείο | Καθοδήγηση βημάτων setup βάσει των οδηγών του μαθήματος | Η **πραγματική εγκατάσταση**, η **σύνδεση VPN**, οι **εκτελέσεις spark-submit** και ο **έλεγχος pods/logs** |
| **Query 3 — κώδικας** (`Q3_df.py`, `Q3_rdd.py`, `Q3_df_joins.py`) | Ίδιο εργαλείο | Βοήθησε στον **αρχικό σκελετό** (SparkSession, paths HDFS, `argparse`, `perf_counter`, βασική ροή read → aggregate → join → write) και στην **ιδέα** χρήσης `reduceByKey` αντί `groupBy` για αποφυγή OOM στο GeoJSON | Η **λογική του Query 3** (τύπος per-capita, parsing income CSV, join analysis με `explain`/`hint`), οι **διορθώσεις μνήμης**, οι **εκτελέσεις benchmark** (3 executors), η **επαλήθευση** 282 ZIP και η **ερμηνεία join strategies** |
| **Αναφορά — Query 3** (`report_query_3.md`) | Ίδιο εργαλείο | Βοήθησε στη **διαμόρφωση κειμένου** και στους πίνακες με βάση τα **δικά μου μετρημένα δεδομένα** | Η **τελική επεξεργασία**, η **επιβεβαίωση αριθμών** από logs/`Q3_times.txt`/`Q3_join_times.txt`, και η **προσωπική ερμηνεία** |
| **Query 4 — κώδικας** (`Q4_df.py`, `Q4_sql.py`, `Q4_df_joins.py`) | Ίδιο εργαλείο | Βοήθησε στον **αρχικό σκελετό** (Haversine σε Spark SQL expressions, cross join, window `row_number`, timing, HDFS paths) | Η **λογική του Query 4** (nearest station, aggregation ανά DIVISION), οι **εκτελέσεις κλιμάκωσης**, η **ανάλυση join** (`explain`/`hint`) και η **επαλήθευση** αποτελεσμάτων DF vs SQL |
| **Αναφορά — Query 4 & ενιαία αναφορά** (`report_full.md`) | Ίδιο εργαλείο | Βοήθησε στη **διαμόρφωση κειμένου**, πινάκων κλιμάκωσης και ενοτήτων join με βάση **μετρημένα δεδομένα** | Η **τελική επεξεργασία**, η **επιβεβαίωση αριθμών** από driver logs/`Q4_times.txt`, και η **προσωπική ερμηνεία** |

## Σημειώσεις

- Η χρήση LLM έγινε **δηλωμένη** σύμφωνα με την εκφώνηση.
- Ο κώδικας εκτελέστηκε στο **lab cluster** και τα αποτελέσματα επαληθεύτηκαν από **driver pod logs** και αρχεία στο `~/bigdata-dsml/results/`.
- Κατανοώ τον κώδικα που παραδίδω και μπορώ να τον εξηγήσω στην προφορική εξέταση.
